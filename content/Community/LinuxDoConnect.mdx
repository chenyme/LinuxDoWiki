# Linux DO Connect

OAuth（Open Authorization），目前最新版本是OAuth 2.0，是一个开放网络标准，旨在实现授权。日常使用的第三方登录方式，如 Google 账号登录，就采用了该标准。OAuth 允许用户授权第三方应用访问存储在其他服务提供商（如 Google）上的信息，而无需在不同平台上重复填写注册表单。用户授权后，平台可以直接访问用户的 Google 账户信息进行身份验证，而用户无需向第三方应用提供密码。

目前系统虽然实现了完整的oauth2 code方式鉴权，不过界面等各种配套还比较简陋，将来会不断完善。来一起打造一个和谐的共享方案吧。

## 基本介绍

这是一套标准的 OAuth2 鉴权，让大家可以共享论坛的用户基本信息。

- 可获取字段：

|参数|说明|
|------|------|
| `id` | 一个不会改变的值 |
| `username` | 论坛用户名 |
| `name` | 论坛的用户昵称，它可能会改变 |
| `avatar_template` | 用户头像的模板URL，可用于生成不同尺寸的头像 |
| `active` | 用户账号是否处于活跃状态 |
| `trust_level` | 0-4 的数值，分别对应着0级到4级 |
| `silenced` | 用户是否被禁言 |
| `external_ids` | 用户的外部ID关联信息 |
| `api_key` | 用户的API访问密钥 |


如果一个公益的网站/接口能够拿到这些数值，那么就可以：

1. 通过 `id` 进行服务频率限制。
2. 通过 `trust_level` 进行服务额度配给。
3. 通过 用户信息 举报滥用。

## 使用说明

### 授权流程

- 用户点击你的应用中的"使用 Linux Do 登录"按钮
- 用户被重定向到 Linux Do 的授权页面
- 用户授权后，Linux Do 将用户重定向回你的应用，并附带授权码
- 你的应用使用授权码获取访问令牌
- 使用访问令牌获取用户信息

### 安全注意事项

- 永远不要在前端暴露你的 Client Secret
- 验证所有从用户那里获取的数据
- 使用 HTTPS 确保数据传输安全
- 定期更新你的 Client Secret
